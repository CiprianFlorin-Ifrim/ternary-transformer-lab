# Ternary Transformer Experiment

A controlled comparison between a standard FP32 transformer and an identical
architecture with ternary-quantized weights ({-1, 0, +1}), trained on four
synthetic mathematical sequence tasks. Both models train simultaneously under
identical conditions so every metric is directly comparable.

---

## Motivation

Ternary (1.58-bit) weights replace floating-point matrix multiplications with
integer addition and subtraction, offering dramatic reductions in memory and
compute at inference time. This experiment studies whether ternary training
dynamics produce competitive accuracy on structured tasks, and how the two
architectures diverge in weight distributions, convergence speed, and per-task
behaviour.

---

## Setup

### 1. Create and activate the conda environment

    conda env create -f environment.yml
    conda activate ternary-transformer

### 2. Register the kernel with Jupyter

    python -m ipykernel install --user --name ternary-transformer --display-name "ternary-transformer"

### 3. Open in VSCode

Select the "ternary-transformer" kernel when prompted.

---

## Tasks

All tasks are serialised as digit token streams (vocabulary size 15). No task
labels are given to the model -- it must internally distinguish tasks from
sequence statistics alone.

| Task       | Description                                                    |
|------------|----------------------------------------------------------------|
| Fibonacci  | Next-token prediction over digit-encoded Fibonacci sequences   |
| FizzBuzz   | Next-token prediction over FizzBuzz output sequences           |
| Parity     | Predict the parity bit at the end of a random bit string       |
| Primes     | Next-token prediction over consecutive prime sequences         |

---

## Architecture

| Dimension     | Value |
|---------------|-------|
| Vocabulary    | 15    |
| Sequence len  | 128   |
| Embed dim     | 32    |
| Heads         | 2     |
| Layers        | 2     |
| FF dim        | 64    |
| Parameters    | ~25k  |
